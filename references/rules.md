## 间距规则 (脚本处理)

### R1a: 中英文之间加空格

来源: 排版指北

```
正确: 使用 GitHub 登录
错误: 使用GitHub登录
```

### R1b: 中文与数字之间加空格

来源: 排版指北

```
正确: 共有 � 个问题
错误: 共有5个问题
```

### R1c: 全角标点与其他字符之间不加空格

来源: 排版指北

```
正确: 你好，世界
错误: 你好 ，世界

正确: 核心思想：简洁
错误: 核心思想 ：简洁
```

## 标点规则 (脚本处理)

### R2a: 中文语境使用全角标点

来源: GB/T 15834

```
正确: 什么是「中文排版」？
错误: 什么是「中文排版」?
```

### R2b: 英文语境使用半角标点

来源: GB/T 15834

```
正确: Use the `format` command.
错误: Use the `format` command。
```

### R2c: 不重复使用标点符号

来源: GB/T 15834

```
正确: 这太棒了！
错误: 这太棒了！！！
```

### R2d: 省略号使用六连点

来源: GB/T 15834

```
正确: 他说……
错误: 他说...
错误: 他说。。。
```

### R2e: 中文括号用全角

来源: GB/T 15834

```
正确: 版本号（v2.0）已发布
错误: 版本号(v2.0)已发布
```

### R2f: 括号配对一致

来源: GB/T 15834

开闭括号必须使用相同宽度。当不一致时，按括号内容判断:
含中文 -> 全角，纯英文/数字 -> 半角。

```
正确: God Object（9152 行）禁止整体输入
错误: God Object(9152 行）禁止整体输入

正确: evidence（TSan 输出），不能仅靠 AI
错误: evidence(TSan 输出），不能仅靠 AI

正确: (Tier 1)
错误: （Tier 1)
```

## Markdown 结构规则 (脚本处理)

### R5a: 标题前后保证空行

来源: CommonMark

```markdown
正确:

前一段落。

## 标题

后一段落。

错误:

前一段落。
## 标题
后一段落。
```

### R5b: 段落之间用空行分隔

来源: CommonMark

```markdown
正确:

第一段内容。

第二段内容。

错误:

第一段内容。
第二段内容。
```

### R6: 列表缩进一致

来源: CommonMark

```markdown
正确:

- 第一项
  - 子项 A
  - 子项 B
- 第二项

错误:

- 第一项
    - 子项 A
  - 子项 B
- 第二项
```

### R7: 表格列对齐

来源: GFM

```markdown
正确:

| 名称   | 说明     |
| ------ | -------- |
| foo    | 示例一   |
| barbaz | 示例二   |

错误:

| 名称 | 说明 |
| --- | --- |
| foo | 示例一 |
| barbaz | 示例二 |
```

## 结构层规则 (Claude 判断)

### S1: 标题层级不跳级

来源: 最佳实践

```markdown
正确:

# 一级
## 二级
### 三级

错误:

# 一级
### 三级 (跳过了二级)
```

### S2: 同级标题不孤立

来源: 最佳实践

同级标题至少出现两个，否则该层级无意义。

```markdown
正确:

## 概述
## 详情

错误:

## 概述 (整篇只有这一个二级标题)
```

### S3: 段落不超过 7 行

来源: 庭说

超过 7 行的段落应拆分为多段或使用列表。

### S4: 同层级列表风格一致

来源: CommonMark

```markdown
正确:

- 项目一
- 项目二
- 项目三

错误:

- 项目一
* 项目二
+ 项目三
```
